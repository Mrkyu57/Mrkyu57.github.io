> 깃허브 블로그를 작성할 때 필요한 Github-Flavored Markdown의 문법

## 머릿말

```markdown
---
title: "글의 제목"
date: XXXX-XX-XX XX:XX:XX +0900
categories: [상위 카테고리, 하위 카테고리]
tags: [태그1, 태그2]
---
```

> 추가적으로, 다음과 같은 설정이 가능합니다
{: .prompt-info }

```markdown
---
title: "글의 제목"
date: XXXX-XX-XX XX:XX:XX +0900
categories: [상위 카테고리, 하위 카테고리]
tags: [태그1, 태그2]
math: true         
toc: true           
pin: true
excerpt: "요약문"
image:
  path: thumbnail.png
  alt: image alternative text
---
```

`math`- 수학기능의 활성화\
`toc`- 게시글의 오른쪽에 표시되는 목록의 표시\
`pin`- 게시글을 블로그 상단에 고정\
`image: path:`- 썸네일 이미지, 1200 x 630 해상도와 1.91 : 1 비율을 권장\
`excerpt`- 외부에서 보이는 글의 요약문

## 제목

```markdown
# 1단계 제목
## 2단계 제목
### 3단계 제목
#### 4단계 제목
##### 5단계 제목
###### 6단계 제목
```

> #과 제목 사이에 반드시 공백이 존재해야 합니다
{: .prompt-tip }

## 목록

```markdown
- 1번째
- 2번째
  - 2.1번째
  - 2.2번째
- 3번쨰
```

> -과 문장 사이에 반드시 공백이 존재해야 합니다
{: .prompt-tip }

> -가 아닌 *를 사용하여도 가능합니다
{: .prompt-info }

## 순서가 있는 목록
```markdown
1. 첫번째
2. 두번째
3. 세번째
5. 네번째 
    5. 다섯째 아님
        6. 여섯째 아님
6. 다섯째
4. 여섯째
```


## 인용문

```markdown
>인용문
```

## 코드 블럭

````markdown
```언어명칭

```
````
